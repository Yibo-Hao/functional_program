# 第二章-纯函数，记忆化，柯里化

# 纯函数概念

相同的输入永远得到相同的输出，没有任何可观察的副作用，纯函数就类似于数学中的函数（用来描述输入和输出的关系）。

# 数组的中的纯函数，非纯函数

  • 纯函数：Array.prototype.slice
  • 非纯函数：Array.prototype.splice

slice() 方法返回一个新的数组对象，这一对象是一个由 begin 和 end 决定的原数组的浅拷贝（包括 begin，不包括end）。原始数组不会被改变。slice实现很简单。如果该参数为负数，则表示从原数组中的倒数第几个元素开始提取，slice(-2) 表示提取原数组中的倒数第二个元素到最后一个元素（包含最后一个元素）。如果省略 begin，则 slice 从索引 0 开始。如果 end 被省略，则 slice 会一直提取到原数组末尾。 如果 end 大于数组的长度，slice 也会一直提取到原数组末尾。

splice() 方法通过删除或替换现有元素或者原地添加新的元素来修改数组,并以数组形式返回被修改的内容。此方法会改变原数组。 array.splice(start[, deleteCount[, item1[, item2[, ...]]]])

细粒度函数非纯繁琐所以我们会使用一些细粒度函数的库例如lodash

# 纯函数的好处-记忆化

由于纯函数对相同的输入始终有相同的结果，所以可以把结果缓存起来，下次相同输入时直接从缓存里查找。

function memoize(fn) {
  let cache = {}
  return function() {
    let key = JSON.stringify(arguments)
    cache[key] = cache[key] || fn.apply(this,arguments)
    return cache[key]
  }
}
  • 纯函数的其他好处-可测试，并行处理
  • 多线程环境下并行操作共享内存数据可能会出现意外情况，纯函数不需要访问共享的内存数据，所以并行环境下可以任意运行纯函数（web worker）
  • 作为纯函数式的语言应该没有“变量(var)”，只有“不变量(val)”，即变量赋值之后不能改变。所以不需要访问共享的内存数据。

# 副作用

副作用让一个函数不纯，如果纯函数依赖外部状态时无法保证输出相同，就会带来副作用。

let mini = 18
function check(age) {
  return age>mini
}
  • 配置文件
  • 全局变量
  • 数据库
  • 用户输入

所有的外部交互都有可能带来副作用，副作用不可能完全禁止，尽量控制副作用在可控范围内发生。

# 科里化（curry）

柯里化其实是函数式编程的一个过程，在这个过程中我们能把一个带有多个参数的函数转换成一系列的嵌套函数。它返回一个新函数，这个新函数期望传入下一个参数。

它不断地返回新函数（像我们之前讲的，这个新函数期望当前的参数），直到所有的参数都被使用。参数会一直保持 alive（通过闭包）这部分参数始终保持不变，当柯里化函数链中最后一个函数被返回和调用的时候，它们会用于执行。

例子：

function match(reg,str) {
  return str.match(reg)
}
match(/\s+/g,"hello")
match(/\d+/g,"abc")
//科里化
const _ = require("lodash")
const match = _.curry(function match(reg,str) {
  return str.match(reg)
})
const = haveSpace = match(/\s+/g)
haveSpace("hello")
const = haveNum = match(/\d+/g)
haveNum("abc")
// 复用 
const filter = _.curry(function(func,array) {
  return array.filter(func)
})
const findSpace = filter(haveSpace)
findSpace(["jone","jone donne"])

可以看到函数复用性得到了史诗级加强

# curry 源代码

function curry(func) {
  return function curredFn(...args){
        if(args.length < func.length){
            return function(){
                return curredFn(...args.concat(Array.from(arguments)))
            }
        }
        return func(...args)
    }
}
  • curry接受一个柯里化的函数记为func，返回一个柯里化之后的函数记为curredFn
  • curredFn是我们真正调用的函数，如果curredFn接受的参数大于等于func的形参个数，那么curredFn直接返回func的调用结果
  • 如果curredFn接受的参数小于func的形参个数，那么返回一个匿名函数递归调用curryFn，此时它的参数拼接了每一个调用的参数
  • 柯里化不可以处理无限个参数函数。

tips：

  • 函数中 arguments.length 指的是实参的个数
  • 函数的 length 指的是形参的个数
  • 浏览器的 length 指浏览器页面和iframe的个数

# 小结

  • 柯里化可以让我们给一个函数传递比较少的参数得到一个已经记住了某些参数的函数
  • 让函数变得更灵活，粒度更小
  • 是一种对函数参数的"缓存"
  • 把多元函数转化成一元函数，使用组合使函数产生强大的功能
