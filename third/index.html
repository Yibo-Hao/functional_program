# 第三章-函数组合

# 函数组合必要性

使用细粒度的函数很容易写出洋葱代码，h(g(z(x))),使用函数组合来避免这种情况出现。函数组合让我们把细粒度函数重新组合成一个新的函数。

# 函数组合（compose）

  • 如果一个函数要经过多个函数处理才能得到最后的值，这个时候可以把中间过程的函数合并成一个函数。
  • 函数就相当于数据的管道，函数组合就相当于把管道连起来
  • 函数组合默认是从右到左执行 fn = compose(f1,f2,f3)
function compose(f, g) {
  return function(value) {
    return f(g(value));
  };
}

lodash 提供了 flow 和 flowRight 两个函数组合

函数组合满足结合律

let f = flowRight(f, g, h);
let f2 = flowRight(flowRight(f, g), h);

# compose 源代码

function compose(...args) {
  return function(value) {
    return args.reverse().reduce((acc, fn) => {
      return fn(acc);
    }, value);
  };
}

# 柯里化，函数组合实战

实现 NEVER SAY DIE 到 never-say-die

const _ = require("lodash");

const spilt = _.curry((sep, str) => _.split(str, sep));
const join = _.curry((sep, array) => _.join(array, sep));
const map = _.curry((fn, array) => _.map(array, fn));

const f = _.flowRight(join("-"), map(_.toLower), spilt(" "));

console.log(f("NEVER SAY DIE"));

把 lodash 的 join 和 map 以及 split 进行改造，虽然它们符合平时的使用，但无法在函数组合中使用。

所以我们进行了柯里化改造。

柯里化常搭配函数组合使用。

# 函数组合的调试

const trace = _.curry((tag, value) => {
  console.log(tag, value);
  return value;
});

在组合函数的后面添加一个 log 函数，记得返回被传递过来的值。trace 函数添加了标记使控制台更容易辨识。

const f = _.flowRight(join("-"), trace("map 后"), map(_.toLower), spilt(" "));

# lodash fp 模块

  • 在 lodash 原始库中，采用数据优先的方法，例如_.map(array,fn)。但是 fp 模块中采用函数优先的方法。
  • lodash fp 模块中 fp.map(fn,array)
  • 此外还有自动科里化的功能，如果你只传入了第一个参数，那么它将返回一个函数
const f = fp.flowRight(
  fp.join("-"),
  fp.map(fp.toLower),
  fp.spilt(" ")
);
